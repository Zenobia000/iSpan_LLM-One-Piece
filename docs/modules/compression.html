核心壓縮技術

模型量化 (Quantization)

降低數值精度以減少記憶體佔用

量化類型

  • 後訓練量化 (PTQ)：GPTQ、AWQ、GGML
  • 量化感知訓練 (QAT)：訓練時模擬量化
  • 動態量化：運行時量化
  • 混合精度：關鍵層保持高精度

精度級別

  • FP16：半精度浮點，2x 壓縮
  • INT8：8位整數，4x 壓縮
  • INT4：4位整數，8x 壓縮
  • Binary：二值化，32x 壓縮
核心演算法：
GPTQ AWQ GGML BitsAndBytes

模型剪枝 (Pruning)

移除不重要的參數和連接

剪枝策略

  • 非結構化剪枝：移除個別權重
  • 結構化剪枝：移除整個通道/層
  • 漸進式剪枝：逐步增加稀疏度
  • 自動剪枝：演算法自動決定

重要性評估

  • 幅度剪枝：基於權重大小
  • 梯度剪枝：基於梯度資訊
  • Fisher 資訊：基於二階導數
  • WANDA：權重與激活感知
前沿方法：
WANDA SparseGPT Lottery Ticket Magnitude

知識蒸餾 (Knowledge Distillation)

將大模型知識轉移到小模型

蒸餾類型

  • 回應蒸餾：學習輸出分佈
  • 特徵蒸餾：學習中間表示
  • 注意力蒸餾：學習注意力模式
  • 關係蒸餾：學習樣本間關係

蒸餾策略

  • 離線蒸餾：預先訓練教師模型
  • 線上蒸餾：同時訓練師生模型
  • 自蒸餾：模型自己當教師
  • 多教師蒸餾：多個教師模型
成功案例：
DistilBERT TinyBERT MobileBERT Alpaca

技術對比分析

壓縮技術選擇決策樹

記憶體極度受限 → INT4量化 或 知識蒸餾
推理速度優先 → 結構化剪枝 + INT8量化
精度要求高 → GPTQ/AWQ量化 或 漸進剪枝
邊緣設備部署 → 蒸餾小模型 + 量化
雲端服務 → 輕度剪枝 + FP16

量化技術

✅ 優勢

  • 實現簡單，工具成熟
  • 壓縮比高，效果顯著
  • 硬體支援好
  • 保持模型結構不變

❌ 劣勢

  • 精度損失不可避免
  • 極端量化效果差
  • 需要校準數據
適用場景：快速部署、硬體受限環境

剪枝技術

✅ 優勢

  • 可控制壓縮程度
  • 理論基礎扎實
  • 可與其他技術結合
  • 推理加速明顯

❌ 劣勢

  • 需要重新訓練
  • 硬體支援有限
  • 結構化剪枝限制多
適用場景：有重訓練資源、追求極致性能

知識蒸餾

✅ 優勢

  • 壓縮比最高
  • 靈活設計學生模型
  • 能力保持相對好
  • 部署資源最少

❌ 劣勢

  • 訓練成本最高
  • 效果不穩定
  • 需要大量數據
  • 能力上限受限
適用場景：邊緣設備、極度資源受限
